This document lists every secret required by the CI workflows in `.github/workflows/` and how to generate them locally.

All secrets are configured under **Settings → Secrets and variables → Actions → New repository secret** in the GitHub repository.

---

## CI flavor triggers

| Trigger | Flavor | Secret prefix |
| --- | --- | --- |
| `workflow_dispatch` (manual) | dev | `DEV_*` |
| `push` to `main` | beta | `BETA_*` |
| prod | _not implemented yet_ | `PROD_*` (reserved) |

---

## Required secrets

| Secret | Format | Target path on runner |
| --- | --- | --- |
| `DEV_ENV_FILE` | raw text | `.env` (overwrites for dev builds) |
| `BETA_ENV_FILE` | raw text | `.env` (overwrites for `beta` builds) |
| `ANDROID_KEY_PROPERTIES` | raw text | `android/key.properties` |
| `ANDROID_KEYSTORE_BASE64` | base64 (no newlines) | `android/app/upload-keystore.jks` |
| `DEV_ANDROID_GOOGLE_SERVICES_JSON_BASE64` | base64 (no newlines) | `android/app/google-services.json` (dev) |
| `BETA_ANDROID_GOOGLE_SERVICES_JSON_BASE64` | base64 (no newlines) | `android/app/google-services.json` (beta) |
| `DEV_IOS_GOOGLE_SERVICES_PLIST_BASE64` | base64 (no newlines) | `ios/Runner/GoogleService-Info.plist` (dev) |
| `BETA_IOS_GOOGLE_SERVICES_PLIST_BASE64` | base64 (no newlines) | `ios/Runner/GoogleService-Info.plist` (beta) |
| `FIREBASE_APP_DIST_SERVICE_ACCOUNT_JSON` | raw JSON (one line) | temp file for Fastlane `firebase_app_distribution` (distribution project `xicarph---builds`) |

### iOS signing (match adhoc)

| Secret | Format | Used for |
| --- | --- | --- |
| `MATCH_PASSWORD` | string | Encrypts/decrypts certs in the match git repo |
| `MATCH_GIT_URL` | HTTPS git URL with embedded PAT | Private match repo, e.g. `https://x-access-token:<PAT>@github.com/org/xicar-ios-signing.git` |
| `MATCH_TEAM_ID` | Apple team ID | Apple Developer team for code signing |

`setup_ci` (called automatically in the iOS CI lane) creates a temporary keychain with a generated password on the runner. No extra GitHub secret is required for keychain access — only the three match secrets above.

Every **build** secret is required for CI runs of the matching flavor. Fastlane fails immediately if a secret is empty.

<Note>
  **Runtime vs distribution Firebase:** `*_ANDROID_GOOGLE_SERVICES_JSON_BASE64` and `*_IOS_GOOGLE_SERVICES_PLIST_BASE64` secrets come from the **runtime** project `xicarv2` (one per flavor). `FIREBASE_APP_DIST_SERVICE_ACCOUNT_JSON` belongs to the **distribution** project `xicarph---builds` and is only used to upload APKs/IPAs to Firebase App Distribution. See [Firebase App Distribution](/ci-cd/firebase-distribution).
</Note>

### Reserved for future prod CI

| Secret | Format | Notes |
| --- | --- | --- |
| `PROD_ENV_FILE` | raw text | `.env` for prod builds (not wired in CI yet) |
| `PROD_ANDROID_GOOGLE_SERVICES_JSON_BASE64` | base64 | Runtime `google-services.json` for prod |
| `PROD_IOS_GOOGLE_SERVICES_PLIST_BASE64` | base64 | Runtime `GoogleService-Info.plist` for prod |

### Optional secrets

| Secret | Format | Used for |
| --- | --- | --- |
| `DISCORD_INTERNAL_SHARING_WEBHOOK_URL` | full Discord webhook URL | Posts Firebase App Distribution release notifications for dev/beta builds |

If `DISCORD_INTERNAL_SHARING_WEBHOOK_URL` is unset, the workflow still succeeds; the Discord step is skipped with a warning.

## Generation

Run these commands from the project root after pulling the latest changes.

### 1. `DEV_ENV_FILE`

Raw `KEY=value` text materialized to `.env` when CI resolves to the dev flavor (`workflow_dispatch`). Copy from your local `.env` with dev values:

```bash
# macOS
cat .env | pbcopy
```

### 2. `BETA_ENV_FILE`

Same format as §1, but with **beta** values (dev backend URLs, beta Firebase keys). Used on push to `main`. Populate a local `.env` with beta content (or maintain a separate untracked file) and copy:

```bash
cat .env | pbcopy   # after filling .env with beta values
```

### 3. `ANDROID_KEY_PROPERTIES`

```properties
storeFile=app/upload-keystore.jks
storePassword=<store-password>
keyAlias=<key-alias>
keyPassword=<key-password>
```

```bash
cat android/key.properties | pbcopy
```

### 4. `ANDROID_KEYSTORE_BASE64`

```bash
base64 -i android/app/upload-keystore.jks | tr -d '\n' | pbcopy
```

### 5. `DEV_ANDROID_GOOGLE_SERVICES_JSON_BASE64`

```bash
base64 -i android/app/google-services.json | tr -d '\n' | pbcopy
```

<Note>
  Use the `google-services.json` content appropriate for the dev flavor (`com.xicartaxi.xicar_user.dev`). The unified multi-client file from Firebase Console also works.
</Note>

### 6. `BETA_ANDROID_GOOGLE_SERVICES_JSON_BASE64`

Same command as §5 — paste into the beta secret. Use beta-appropriate content if you maintain separate files locally.

### 7. `DEV_IOS_GOOGLE_SERVICES_PLIST_BASE64`

```bash
base64 -i ios/Runner/GoogleService-Info.plist | tr -d '\n' | pbcopy
```

<Note>
  Use the plist content appropriate for the dev flavor (`com.xicar.userapp.dev`).
</Note>

### 8. `BETA_IOS_GOOGLE_SERVICES_PLIST_BASE64`

Same command as §7 — paste into the beta secret.

### 9. `FIREBASE_APP_DIST_SERVICE_ACCOUNT_JSON`

Service account JSON for the **distribution** project `xicarph---builds` (not `xicarv2`).

1. Google Cloud Console → project `xicarph---builds` → **IAM & Admin** → **Service accounts** → create `ci-app-distribution`
2. Grant **Firebase App Distribution Admin** (`roles/firebaseappdist.admin`)
3. **Keys** → **Add key** → **JSON** → download
4. Paste the entire JSON into the GitHub secret

```bash
cat /path/to/xicarph-builds-sa.json | pbcopy
```

### 10. iOS match secrets

One-time local bootstrap — see [iOS CI](/ci-cd/ios). Run `bundle exec fastlane ios sync_signing` locally (registers Developer-portal App IDs via `produce --skip_itc`, then match adhoc profiles). Only use `produce` **without** `skip_itc` if you intentionally need an App Store Connect app record.

| Secret | How to obtain |
| --- | --- |
| `MATCH_PASSWORD` | Choose a strong passphrase; store in 1Password |
| `MATCH_GIT_URL` | `https://x-access-token:<GITHUB_PAT>@github.com/org/xicar-ios-signing.git` (PAT needs repo read) |
| `MATCH_TEAM_ID` | Apple Developer → Membership → Team ID |

## Local files that must stay gitignored

- `.env`
- `android/key.properties`
- `android/app/upload-keystore.jks`
- `android/app/google-services.json`
- `ios/Runner/GoogleService-Info.plist`

```bash
git check-ignore -v .env android/key.properties android/app/upload-keystore.jks android/app/google-services.json ios/Runner/GoogleService-Info.plist
```

## Rotation checklist

Trigger via **Actions → Build (Android \+ iOS) → Run workflow**.

- **Keystore rotated** → `ANDROID_KEYSTORE_BASE64` \+ `ANDROID_KEY_PROPERTIES`
- **Runtime Firebase changed** → regenerate the flavor-specific `*_ANDROID_GOOGLE_SERVICES_JSON_BASE64` and/or `*_IOS_GOOGLE_SERVICES_PLIST_BASE64` secret
- **`.env` values changed** → regenerate the flavor-specific `*_ENV_FILE` secret
- **Distribution SA compromised** → revoke old key in GCP, regenerate `FIREBASE_APP_DIST_SERVICE_ACCOUNT_JSON`
- **Match certs expired** → run `bundle exec fastlane match adhoc --force` locally, commit to match repo
