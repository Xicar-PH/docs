Use this guide when you join a Xicar development project.

## Before you begin

You need:

- A GitHub account
- An invitation to the [Xicar GitHub organization](https://github.com/xicar-ph)
- Git installed on your computer
- The tools required by the repository you are working on, such as Flutter or Laravel

Access to Bitwarden, Firebase, and Hostinger is also invite-only. Ask for access when your work requires these services.

## Set up a repository

<Steps>
  <Step title="Get access">
    Confirm that you can access the [Xicar GitHub organization](https://github.com/xicar-ph). If you need access to GitHub, Bitwarden, Firebase, or Hostinger, please reach out.
  </Step>
  <Step title="Clone the repository">
    Ask a teammate which repository you should work on. Then clone it with Git:

    ```bash
    git clone https://github.com/xicar-ph/<repository-name>.git
    cd <repository-name>
    ```

    Replace `<repository-name>` with the repository name you were given.
  </Step>
  <Step title="Follow the repository setup">
    Read the repository's README for its required versions, dependencies, environment variables, and run commands.

    The setup may differ between repositories. For example, a Flutter repository and a Laravel repository have different local requirements.
  </Step>
  <Step title="Run and verify locally">
    Start the project using the commands documented in its README. Confirm that it runs locally before making changes.
  </Step>
</Steps>

## Service access

Use these services only when your repository or task requires them:

- **Bitwarden** stores shared credentials and secrets.
- **Firebase** provides application services and distribution tools.
- **Hostinger** provides hosting.

Do not request or share credentials in GitHub issues, pull requests, or chat. Use Bitwarden for shared secrets and ask a teammate if you are unsure where a credential belongs.
